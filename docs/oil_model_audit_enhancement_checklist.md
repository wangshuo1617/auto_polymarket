# Oil 模型审计增强清单

目标：在“消息面强影响”的前提下，提升 Oil 分析链路的可审计性、稳健性和可复盘性。  
范围：`position_analyze_oil.py`、`ai/researcher.py`、`ai/prompts.py`、日志与报告输出。

---

## 当前状态快照（基线）

- [x] Oil 分析使用 grounding 检索新闻
- [x] 报告会引用上一轮 `last_report_oil.json`
- [ ] 未落盘 grounding 来源（URL、标题、抓取时间、查询词）
- [ ] 未输出“证据置信度/来源一致性”评分
- [ ] API 503 时无降级策略（本轮直接失败）
- [ ] 无“消息面反转”回放测试

---

## P0（必须优先，先做这 4 项）

### 1) 来源证据落盘（Source Traceability）

- [ ] 在 Oil 分析结果中新增 `news_sources` 字段（数组）
- [ ] 每条来源至少包含：`url`、`title`、`fetched_at`、`query_tag`
- [ ] 每轮将来源写入本地审计文件（建议 `logs/oil_news_sources.jsonl`）

**验收标准**

- [ ] 任意一封 Oil 邮件报告都能回溯到 >= 3 条来源
- [ ] 审计文件可按时间检索，支持追溯某次建议对应来源

---

### 2) 证据置信度闸门（Confidence Gate）

- [ ] 在模型输出 schema 中增加：
  - [ ] `news_confidence`（0-100）
  - [ ] `source_count`
  - [ ] `source_consistency`（high/medium/low）
- [ ] 增加执行闸门规则：
  - [ ] `news_confidence < 60` 时，禁止新增激进建仓建议
  - [ ] `source_count < 3` 时，`执行优先级` 最高只能是“挂单等待”
  - [ ] `source_consistency=low` 时，强制附加“观望/减仓”提示

**验收标准**

- [ ] 低置信度样本下，报告不再出现“立即执行 + 大额新仓”
- [ ] 同样行情下，多次运行建议波动减小（人工抽检 10 次）

---

### 3) 503/超时降级策略（Fail-Safe）

- [ ] 为 Oil 模型调用增加重试与指数退避
- [ ] 重试失败后使用降级路径：
  - [ ] 回退到“规则模板报告”（基于 K 线+持仓+风控）
  - [ ] 标记 `analysis_mode=fallback`
  - [ ] 明确提示“本轮未完成新闻检索”
- [ ] 保障任务退出码为成功（避免调度链中断）

**验收标准**

- [ ] 人为制造 API 不可用时，调度仍持续运行
- [ ] 失败轮次也会产出可读报告，并清晰标注降级模式

---

### 4) 报告去锚定（Previous Report Anti-Anchoring）

- [ ] 保留 `previous_report` 作为参考，但新增提示约束：
  - [ ] 必须先给“本轮新证据摘要”再给结论
  - [ ] 若与上轮结论冲突，必须解释冲突原因
- [ ] 增加字段 `delta_from_prev`：本轮与上轮建议变化摘要

**验收标准**

- [ ] 连续两轮建议变化时，能清楚看到“为何改变”
- [ ] 避免无新证据情况下的方向大跳变

---

## P1（建议尽快，增强稳定性）

### 5) 消息面结构化输入

- [ ] 将“新闻摘要”与“行情特征”分离输入，避免混写
- [ ] 每条新闻摘要结构化：`event`、`impact_direction`、`impact_horizon`、`confidence`

**验收标准**

- [ ] 模型输出可区分“短线冲击”与“月内方向”

---

### 6) 建议强度限幅（Position Sizing Guardrail）

- [ ] 当 `news_confidence` 低/冲突高时，限制单标的新增仓位上限
- [ ] 当 `ATR%` 超高时，自动降低新增仓位比例（波动放大保护）

**验收标准**

- [ ] 高波动+低证据时，仓位建议自动收缩

---

### 7) 审计可视化汇总（日报）

- [ ] 生成每日审计摘要（建议 `output/oil_audit_daily_YYYY-MM-DD.md`）
- [ ] 内容包括：来源数量、置信度分布、降级次数、建议方向变更次数

**验收标准**

- [ ] 可 5 分钟内完成“今天为何看多/看空”的复盘

---

## P2（回测与治理）

### 8) 反转场景回放测试（News Flip Replay）

- [ ] 构造至少 20 组“多头新闻 -> 空头新闻”的回放样本
- [ ] 检查建议是否存在过度追涨杀跌

**验收标准**

- [ ] 回放中建议方向切换不超过设定阈值（例如 24h 内 <= 2 次）

---

### 9) 审计指标（KPI）固化

- [ ] 固化关键指标：
  - [ ] `source_coverage_rate`
  - [ ] `fallback_rate`
  - [ ] `recommendation_flip_rate`
  - [ ] `high_confidence_action_rate`

**验收标准**

- [ ] 每周可导出 KPI 并与上周对比

---

## 实施顺序建议（最短路径）

1. 先做 P0-1/2/3（来源落盘、置信度闸门、503降级）  
2. 再做 P0-4（去锚定）  
3. 然后补 P1（强度限幅 + 日报）  
4. 最后做 P2（回放与 KPI 治理）

---

## 完成定义（DoD）

- [ ] 发生 API 503 时系统不再“无报告失败”
- [ ] 任意建议均可回溯证据来源
- [ ] 低置信度情况下不会触发激进执行建议
- [ ] 可复盘“本轮建议为何变化”
