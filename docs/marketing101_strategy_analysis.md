### 2.1 核心逻辑：BTC 动量跟随

marketing101 的方向选择高度依赖 **BTC 实时价格相对窗口开盘价的偏离**：

- **80.1% 的仓位跟随 BTC 当前方向**（BTC > 开盘 → 买 Up，BTC < 开盘 → 买 Down）
- 仅 19.9% 的仓位逆 BTC 方向入场

这不是随机的 50/50 猜测，而是一套**趋势确认后跟随**的策略。

### 2.2 BTC 偏离幅度与胜率的关系（核心发现）

通过交叉 btc_poly_1s_ticks 数据库中的 BTC 实时价格，得到以下关键关系：

| BTC 偏离开盘价 | 样本数 | 跟随方向胜率 | 入场 Token 均价 | 平均入场时间 |
|---------------|--------|------------|----------------|------------|
| $5 – $20 | 50 | 54% | 0.54 | 92s |
| $20 – $40 | 91 | 66% | 0.63 | 153s |
| **$40 – $60** | **71** | **82%** | **0.68** | **152s** |
| $60 – $100 | 71 | 79% | 0.78 | 162s |
| $100+ | 30 | 97% | 0.84 | 125s |

**关键洞察**：BTC 在 5 分钟窗口内偏离越大，越倾向于保持该方向直到结算。当 BTC 偏离 ≥$40 时，跟随方向的胜率跃升到 80%+ 。

### 2.3 最优入场区间

通过网格搜索 BTC 偏离幅度 × 入场时间的组合：

| BTC 偏离 ≥ | 入场时间 | 样本数 | 胜率 | PnL |
|-----------|---------|--------|------|-----|
| $40 | 60–120s | 44 | 86% | +$11,577 |
| $50 | 120–180s | 35 | 86% | +$2,283 |
| $20 | 120–180s | 64 | 84% | +$26,585 |
| $40 | 120–180s | 45 | 84% | +$5,868 |
| $50 | 180–300s | 52 | 85% | +$6,747 |

**最佳甜蜜点：BTC 偏离 ≥$40，入场时间 60–180 秒，胜率 84–86%。**

此时 Token 价格通常在 0.63–0.68 区间，赢利空间（0.32–0.37）远大于亏损空间（0.63–0.68），提供了正期望值。

### 2.4 入场时间分布

入场时间在 0–300 秒内近似均匀分布，中位数 129 秒：

| 时间段 | 仓位数 | 跟随 BTC 比例 |
|--------|--------|-------------|
| 0–30s | 50 | 68% |
| 30–60s | 37 | 76% |
| 60–120s | 98 | 81% |
| 120–180s | 85 | 80% |
| 180–240s | 68 | 87% |
| 240–300s | 55 | 87% |

**越晚入场，跟随 BTC 的比例越高**。早期入场（0–30s）方向还不明朗，跟随率仅 68%；后期入场（180s+）方向已确立，跟随率达 87%。

与此对应，Token 价格也呈现**时间-价格递增**关系：

| 入场时间 | Token 入场均价 |
|---------|--------------|
| 0–30s | ~0.50 |
| 60–120s | ~0.60 |
| 180–240s | ~0.76 |

早期入场虽然价格便宜（利润空间大），但方向不确定性高；晚期入场方向更确定，但价格已贵（利润空间小）。marketing101 在两端都有分布，说明他会根据 BTC 信号强弱灵活选择入场时机。

### 2.5 方向选择的连续性

- **59.7% 的交易与前一笔方向相同**
- 最长连续同方向：22 次
- 整体方向中性：243 Up vs 239 Down（50.4% / 49.6%）

这表明他不是交替押注，而是**在趋势持续时连续跟随同一方向**。

| 前 3 笔结果 | 后续平均仓位 | 样本数 |
|------------|-----------|--------|
| 连续 3 胜 | $372 | 111 次 |
| 混合结果 | $332 | 332 次 |
| **连续 3 败** | **$194** | **36 次** |

**连续 3 败后仓位缩减至正常的 ~50%**，是经典的"输了缩仓"风控策略，避免在不利周期中扩大损失。

### 3.6 窗口内仓位控制

> **修正**：原始 closed-positions 分析未检测到双向窗口，但 BUY 交易明细（114 窗口样本）显示 **21 个窗口（18%）同时买了 Up 和 Down**。

这些双向窗口分为三类：
- **一侧可忽略（<$20）**：6 个 — 试探性小单，不构成真正的双向仓位
- **先后换方向（无时间重叠）**：11 个 — 先买 A 方向，方向判断改变后买 B 方向（如先在 77s 买 $93 Up，后在 303s 买 $391 Down）
- **时间重叠双向**：4 个 — 同时持有两个方向的仓位

**主要模式是"换方向"而非"对冲"**——当 BTC 方向反转时，他会切换到新方向下注，但不一定卖掉原方向仓位（因为卖出有 spread 成本）。82% 的窗口仍然只有单方向仓位。

## 四、退出策略

### 4.1 100% 持有至结算

在全部 1,657 个仓位中：

- **卖出笔数：0**
- **所有仓位都持有到 5 分钟窗口结算**

这是最简单也最极端的退出策略——**完全不做止损、止盈或中途调整**。

### 4.2 为什么不卖？

这与他的入场逻辑和仓位管理形成闭环：

1. **入场时已通过 BTC 偏离确认方向**——他认为入场时的判断大概率正确（甜蜜区 84% 胜率），不需要中途调整
2. **仓位管理已内置风控**——小仓试探 + 确认后加码，错误仓位本身就是小仓（$48），即使全亏损失也可控
3. **5 分钟窗口极短**——市场微观结构决定了中途卖出的滑点和 spread 成本可能吞噬大部分利润
4. **避免情绪化操作**——100% 持有到结算意味着不需要做任何卖出决策，减少了"恐慌卖出"的可能

### 4.3 退出策略的风险

- **无止损**意味着每个错误仓位都是**全额亏损**（Token 结算为 0，损失整个仓位成本）
- 这要求入场胜率必须足够高且仓位管理足够严格来覆盖亏损
- 计算：在入场价 0.65 时，胜率需 ≥ 65% 才能盈亏平衡（他在甜蜜区达到 82–86%）

## 5.5 Underdog（逆势彩票）策略

m101 在约 12%（106/845）的窗口中执行一种**独立的 underdog 策略**：以极低价格买入 BTC 反方向的 token，押注 5 分钟内 BTC 趋势反转。

### 5.5.1 策略定义

| 特征 | 数值 |
|------|------|
| 窗口数（纯 underdog） | 106（12.5%） |
| 双向窗口中的彩票侧 | 另有 66 个窗口的次要方向 avg < 0.30 |
| 总计 | 172 个窗口涉及 underdog 操作 |
| 方向选择 | **83% 逆 BTC 当前趋势**（BTC 涨 → 买 Down，BTC 跌 → 买 Up）|
| 平均买入价 | 0.14（纯 underdog） |
| 胜率 | 21.2%（纯），19.4%（双向彩票侧） |
| ROI | **+37.5%**（纯），+47.1%（双向彩票侧） |
| 总 PnL | +$6,318（纯 underdog） |

### 5.5.2 触发条件分析

**高波动是必要条件，但不充分**：

| BTC 最大偏离 | 所有窗口 | 其中 underdog | 占比 |

| ≥ $100 | 372 | 74 | 20% |

| ≥ $250 | 88 | 21 | 24% |

Underdog 窗口 BTC 最大偏离 avg=$199（vs 非 underdog avg=$114），但 80% 的高波动窗口 m101 仍选择跟随方向而非逆势。

**入场时 BTC 已偏离的幅度**：avg=$115，median=$94。m101 在 BTC 已大幅偏离后、判断可能过度延伸时入场。

### 5.5.5 Maker 优势在 Underdog 中尤为关键

m101 实际支付的价格比市场 ask 平均**便宜 $0.167**。在 avg ask 仅 0.31 的 underdog token 上，这意味着 maker 挂单拿到了**近乎半价**的成交。对于 taker 执行，underdog 策略的 EV 会大幅下降。

### 5.5.6 BTC 实际反转率

仅 **16%** 的 underdog 窗口中 BTC 最终反转（结算方向与入场时 BTC 趋势相反）。这与 21% 的胜率一致——underdog 策略本质是**低概率高赔率**：

- 价格 < 0.10：214% ROI（赔率 10:1，只需 >10% 胜率即正 EV）
- 价格 0.10-0.15：44% ROI
- 价格 0.20-0.30：**-22% ROI**（赔率不够高，负 EV）

### 5.5.7 对我们策略的启示

| 结论 | 说明 |
|------|------|
| ⚠️ **需要 Maker 才有 EV** | Taker 买入 underdog token 的成本比 m101 高约 50%，可能使正 EV 变为负 |
| ✅ **甜蜜区：price < 0.15** | 只有极低价格的 token 才有足够高的赔率覆盖低胜率 |
| ✅ **小额投入** | 每次 $50-100 的彩票注，不影响主策略资金 |
| ⛔ **暂不实现** | 依赖 maker 执行 + 未知的窗口选择信号，复制性低（⭐⭐）|

## 六、策略总结

### 完整交易流程

```
窗口开始 (0s)
  │
  ├─ 观察 BTC 实时价格 vs 窗口开盘价
  │
  ├─ 等待 BTC 偏离 ≥ $40（最优 $40–$100）
  │     └─ 如果始终不偏离 → 可能不入场或小仓试探
  │
  ├─ 确认方向：BTC > 开盘 → 买 Up，BTC < 开盘 → 买 Down（80% 概率跟随）
  │
  ├─ 分层挂限价单：在目标 token 上挂出多个价位的买单（avg 44 笔/窗口）
  │     └─ 挂单价格比 taker 平均便宜 0.008
  │     └─ 持续挂单 88–133 秒，覆盖窗口大部分时间
  │     └─ 占总投入的 68%（$631K）
  │
  ├─ Taker 补仓：在方向确认或需要快速建仓时用 taker 吃单（占 32%）
  │     └─ taker 单笔更大（avg $61 vs maker $19），更像"确认性加仓"
  │     └─ 11.4% 的窗口只用 taker（可能是紧急/高确定性场景）
  │
  ├─ 可选 DCA：36% 的窗口有多次入场决策（中位间隔 18s，最多 8 次）
  │     └─ 追涨/补仓各占约 50%
  │     └─ 高确定性信号 → 重仓（$1,000+），低确定性 → 轻仓
  │
  ├─ 方向修正：18% 的窗口会切换方向（BTC 反转时买入对面 token）
  │
  ├─ 连败风控：连续 3 败后仓位缩减 ~50%
  │
  └─ 持有至结算（300s），从不中途卖出
```

### 盈利来源

1. **BTC 动量持续性**：BTC 在 5 分钟窗口内偏离 $40+ 后，有 80%+ 的概率保持该方向到结算
2. **仓位管理的非对称性**：高确定性信号下重仓（82% 胜率 × $1,440 均仓），低确定性轻仓（48% 胜率 × $22 均仓）
3. **Token 价格的有利入场点**：甜蜜区入场价 0.63–0.68，赢利空间（0.32–0.37）大于亏损空间
4. **Maker 挂单优势**：限价单拿到比市价更好的成交价（avg 便宜 0.008），降低了入场成本

### 策略风险

1. **无止损**：错误仓位全额亏损。需要持续保持 >60% 的胜率才能盈利
2. **单日集中度**：04-13 单日贡献 44% 的总 PnL（$60,908），存在**利润集中在少数大波动日**的风险
3. **市场结构变化**：如果 BTC 5 分钟内动量持续性降低（例如做市商改进算法），策略边际可能消失
4. **样本有限**：21 天数据，策略长期稳定性未知

### 可复制性评估

| 维度 | 评估 |
|------|------|
| 信号明确性 | ⭐⭐⭐⭐⭐ BTC 实时价格 vs 开盘价，完全可量化 |
| 入场规则 | ⭐⭐⭐⭐ 偏离阈值 + 时间窗口 + 方向跟随，可编程 |
| 入场执行 | ⭐⭐⭐⭐ m101 用 87% maker 挂单拿到更优价格（avg 便宜 0.008），但用 taker 执行同样可行，maker 优势是锦上添花而非核心 alpha |
| 仓位管理 | ⭐⭐⭐⭐ 64% 单次入场，36% 有 DCA（间隔中位 18s），逻辑相对清晰 |
| 退出规则 | ⭐⭐⭐⭐⭐ 100% 持有到结算，最简单的规则 |
| 方向修正 | ⭐⭐⭐ 18% 窗口会换方向，触发条件不完全透明 |
| 整体可复制性 | ⭐⭐⭐⭐ 核心 alpha 来自方向判断+仓位管理，用 taker 执行即可复制；maker 挂单是执行层优化，对小资金影响有限 |
