## 二、入场策略

### 2.1 核心逻辑：BTC 动量跟随

marketing101 的方向选择高度依赖 **BTC 实时价格相对窗口开盘价的偏离**：

- **80.1% 的仓位跟随 BTC 当前方向**（BTC > 开盘 → 买 Up，BTC < 开盘 → 买 Down）
- 仅 19.9% 的仓位逆 BTC 方向入场

这不是随机的 50/50 猜测，而是一套**趋势确认后跟随**的策略。

### 2.2 BTC 偏离幅度与胜率的关系（核心发现）

通过交叉 btc_poly_1s_ticks 数据库中的 BTC 实时价格，得到以下关键关系：

| BTC 偏离开盘价 | 样本数 | 跟随方向胜率 | 入场 Token 均价 | 平均入场时间 |
|---------------|--------|------------|----------------|------------|
| $5 – $20 | 50 | 54% | 0.54 | 92s |
| $20 – $40 | 91 | 66% | 0.63 | 153s |
| **$40 – $60** | **71** | **82%** | **0.68** | **152s** |
| $60 – $100 | 71 | 79% | 0.78 | 162s |
| $100+ | 30 | 97% | 0.84 | 125s |

**关键洞察**：BTC 在 5 分钟窗口内偏离越大，越倾向于保持该方向直到结算。当 BTC 偏离 ≥$40 时，跟随方向的胜率跃升到 80%+ 。

### 2.3 最优入场区间

通过网格搜索 BTC 偏离幅度 × 入场时间的组合：

**最佳甜蜜点：BTC 偏离 ≥$40，入场时间 60–180 秒，胜率 84–86%。**

此时 Token 价格通常在 0.63–0.68 区间，赢利空间（0.32–0.37）远大于亏损空间（0.63–0.68），提供了正期望值。

### 2.4 入场时间分布

入场时间在 0–300 秒内近似均匀分布，中位数 129 秒：

| 时间段 | 仓位数 | 跟随 BTC 比例 |
|--------|--------|-------------|
| 0–30s | 50 | 68% |
| 30–60s | 37 | 76% |
| 60–120s | 98 | 81% |
| 120–180s | 85 | 80% |
| 180–240s | 68 | 87% |
| 240–300s | 55 | 87% |

**越晚入场，跟随 BTC 的比例越高**。早期入场（0–30s）方向还不明朗，跟随率仅 68%；后期入场（180s+）方向已确立，跟随率达 87%。

与此对应，Token 价格也呈现**时间-价格递增**关系：

| 入场时间 | Token 入场均价 |
|---------|--------------|
| 0–30s | ~0.50 |
| 60–120s | ~0.60 |
| 180–240s | ~0.76 |

早期入场虽然价格便宜（利润空间大），但方向不确定性高；晚期入场方向更确定，但价格已贵（利润空间小）。marketing101 在两端都有分布，说明他会根据 BTC 信号强弱灵活选择入场时机。

### 2.5 方向选择的连续性

- **59.7% 的交易与前一笔方向相同**
- 最长连续同方向：22 次
- 整体方向中性：243 Up vs 239 Down（50.4% / 49.6%）

这表明他不是交替押注，而是**在趋势持续时连续跟随同一方向**。

### 3.6 窗口内仓位控制

**每个 5 分钟窗口最多持有 1 个方向的仓位**——在 482 个样本中，0 个窗口出现双向对冲。这说明他每个窗口只下一个方向的注，不做对冲。

## 四、退出策略

### 4.1 100% 持有至结算

在全部 1,657 个仓位中：

- **卖出笔数：0**
- **所有仓位都持有到 5 分钟窗口结算**

这是最简单也最极端的退出策略——**完全不做止损、止盈或中途调整**。

### 4.2 为什么不卖？

这与他的入场逻辑和仓位管理形成闭环：

1. **入场时已通过 BTC 偏离确认方向**——他认为入场时的判断大概率正确（甜蜜区 84% 胜率），不需要中途调整
2. **仓位管理已内置风控**——小仓试探 + 确认后加码，错误仓位本身就是小仓（$48），即使全亏损失也可控
3. **5 分钟窗口极短**——市场微观结构决定了中途卖出的滑点和 spread 成本可能吞噬大部分利润
4. **避免情绪化操作**——100% 持有到结算意味着不需要做任何卖出决策，减少了"恐慌卖出"的可能

### 4.3 退出策略的风险

- **无止损**意味着每个错误仓位都是**全额亏损**（Token 结算为 0，损失整个仓位成本）
- 这要求入场胜率必须足够高且仓位管理足够严格来覆盖亏损
- 计算：在入场价 0.65 时，胜率需 ≥ 65% 才能盈亏平衡（他在甜蜜区达到 82–86%）

## 六、策略总结

### 完整交易流程

```
窗口开始 (0s)
  │
  ├─ 观察 BTC 实时价格 vs 窗口开盘价
  │
  ├─ 等待 BTC 偏离 ≥ $40（最优 $40–$100）
  │     └─ 如果始终不偏离 → 可能不入场或小仓试探
  │
  ├─ 确认方向：BTC > 开盘 → 买 Up，BTC < 开盘 → 买 Down（80% 概率跟随）
  │
  ├─ 首次入场：小额买入（中位 $48/笔）
  │     └─ 最优入场时间：60–180 秒
  │
  ├─ DCA 加仓：方向持续确认 → 继续买入（中位 3 次，最多 40 次）
  │     └─ 61% 追涨加仓，39% 逢低补仓
  │     └─ 高确定性信号 → 重仓（$1,000+），低确定性 → 轻仓（$22）
  │
  ├─ 连败风控：连续 3 败后仓位缩减 ~50%
  │
  └─ 持有至结算（300s），从不中途卖出
```

### 盈利来源

1. **BTC 动量持续性**：BTC 在 5 分钟窗口内偏离 $40+ 后，有 80%+ 的概率保持该方向到结算
2. **仓位管理的非对称性**：高确定性信号下重仓（82% 胜率 × $1,440 均仓），低确定性轻仓（48% 胜率 × $22 均仓）
3. **Token 价格的有利入场点**：甜蜜区入场价 0.63–0.68，赢利空间（0.32–0.37）大于亏损空间

### 策略风险

1. **无止损**：错误仓位全额亏损。需要持续保持 >60% 的胜率才能盈利
2. **单日集中度**：04-13 单日贡献 44% 的总 PnL（$60,908），存在**利润集中在少数大波动日**的风险
3. **市场结构变化**：如果 BTC 5 分钟内动量持续性降低（例如做市商改进算法），策略边际可能消失
4. **样本有限**：21 天数据，策略长期稳定性未知

### 可复制性评估

| 维度 | 评估 |
|------|------|
| 信号明确性 | ⭐⭐⭐⭐⭐ BTC 实时价格 vs 开盘价，完全可量化 |
| 入场规则 | ⭐⭐⭐⭐ 偏离阈值 + 时间窗口 + 方向跟随，可编程 |
| 仓位管理 | ⭐⭐⭐ DCA 频率和金额的动态调整逻辑不完全透明 |
| 退出规则 | ⭐⭐⭐⭐⭐ 100% 持有到结算，最简单的规则 |
| 整体可复制性 | ⭐⭐⭐⭐ 核心逻辑可复制，DCA 细节需要进一步校准 |
