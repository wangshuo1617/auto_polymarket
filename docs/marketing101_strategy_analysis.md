## 二、入场策略

### 2.0 Maker/Taker 执行模式

m101 以大量分层限价单（maker）为主，辅以少量 taker 吃单：

| 指标 | Taker（吃单） | Maker（挂单） |

**Maker 执行是 m101 的核心 alpha 来源**——以低于 market ask 0.05-0.10 的价格成交（在低价 token 上优势更大），这是 m101 实现正 EV 的关键。

### 2.0.5 推测的入场流程

```
窗口开始 (0s)
  │
  ├─ 观察 BTC 方向信号（纯 BTC 价格 vs 开盘价，无隐藏微结构信号）
  │
  ├─ 在目标方向的 token 上挂出多个价位的限价买单（分层挂单）
  │     └─ avg 44 笔/窗口，持续 88-133s
  │     └─ 价格比 market ask 便宜 0.05-0.10
  │
  ├─ 部分限价单被其他交易者的卖单成交（maker fill）
  │
  ├─ 在方向确认或需要快速建仓时，用 taker 市价吃单补仓
  │     └─ taker 单笔更大（avg $61 vs maker $19）
  │     └─ 96 个窗口只用 taker（11.4%）
  │
  └─ 持有至结算，从不卖出
```

### 2.1 核心逻辑：BTC 动量跟随

marketing101 的方向选择高度依赖 **BTC 实时价格相对窗口开盘价的偏离**：

- **92% 的非 underdog 仓位跟随 BTC 当前方向**（BTC > 开盘 → 买 Up，BTC < 开盘 → 买 Down）
- 整体（含 underdog）约 80% 跟随——因为 underdog 策略是刻意逆向的
- **控制变量实验证明 m101 不使用 BTC 价格之外的隐藏信号**（orderbook depth、资金费率等不影响准确率）

### 2.2 BTC 偏离幅度与胜率的关系（核心发现）

通过交叉 btc_poly_1s_ticks 数据库中的 BTC 实时价格，得到以下关键关系：

| BTC 偏离开盘价 | 样本数 | 跟随方向胜率 | 入场 Token 均价 | 平均入场时间 |
|---------------|--------|------------|----------------|------------|
| $5 – $20 | 50 | 54% | 0.54 | 92s |
| $20 – $40 | 91 | 66% | 0.63 | 153s |
| **$40 – $60** | **71** | **82%** | **0.68** | **152s** |
| $60 – $100 | 71 | 79% | 0.78 | 162s |
| $100+ | 30 | 97% | 0.84 | 125s |

**关键洞察**：BTC 在 5 分钟窗口内偏离越大，越倾向于保持该方向直到结算。当 BTC 偏离 ≥$40 时，跟随方向的胜率跃升到 80%+ 。

### 2.3 入场价格分析

m101 的非 underdog 入场价格按价格桶分布：

| 价格桶 | 窗口数 | 平均入场价 | BTC 偏离 | 入场时间 |
|-------|--------|----------|---------|---------|
| <0.50 | 144 | 0.422 | $29 | 48s |
| 0.50-0.65 | 141 | 0.576 | $34 | 83s |
| 0.65-0.75 | 91 | 0.700 | $48 | 107s |
| 0.75-0.85 | 84 | 0.800 | $59 | 123s |
| >=0.85 | 114 | 0.903 | $98 | 168s |

> ⚠️ **Polymarket 定价高效**：虽然低 BTC 偏离→低入场价→看似更大利润空间，但 token 价格精确反映了真实概率。在任意 BTC 偏离阈值下以 taker 入场，**E[ROI] 都接近零**（详见§六信号源分析）。m101 的正 EV 来自 maker 执行以低于 market ask 的价格成交，而非入场时机选择。

### 2.4 入场时间分布

入场时间在 0–300 秒内近似均匀分布，中位数 129 秒：

| 时间段 | 仓位数 | 跟随 BTC 比例 |
|--------|--------|-------------|
| 0–30s | 50 | 68% |
| 30–60s | 37 | 76% |
| 60–120s | 98 | 81% |
| 120–180s | 85 | 80% |
| 180–240s | 68 | 87% |
| 240–300s | 55 | 87% |

**越晚入场，跟随 BTC 的比例越高**。早期入场（0–30s）方向还不明朗，跟随率仅 68%；后期入场（180s+）方向已确立，跟随率达 87%。

与此对应，Token 价格也呈现**时间-价格递增**关系：

| 入场时间 | Token 入场均价 |
|---------|--------------|
| 0–30s | ~0.50 |
| 60–120s | ~0.60 |
| 180–240s | ~0.76 |

早期入场虽然价格便宜（利润空间大），但方向不确定性高；晚期入场方向更确定，但价格已贵（利润空间小）。marketing101 在两端都有分布，说明他会根据 BTC 信号强弱灵活选择入场时机。

### 2.5 方向选择的连续性

- **59.7% 的交易与前一笔方向相同**
- 最长连续同方向：22 次
- 整体方向中性：243 Up vs 239 Down（50.4% / 49.6%）

这表明他不是交替押注，而是**在趋势持续时连续跟随同一方向**。

### 2.6 逆 BTC 入场分析

整体约 20% 的仓位逆 BTC 方向入场，其中绝大部分属于 **Underdog 彩票策略**（详见 §5.5）：

| 类别 | 仓位数 | 胜率 | 说明 |
|------|--------|------|------|
| Underdog 彩票（price<0.30） | ~106 | 21% | 刻意逆向的独立策略 |
| 非 underdog 逆向 | ~40 | ~33% | 早期方向未定时的误判 |

非 underdog 的逆向入场集中在窗口早期（0-60s 占 35%），投入极小（中位 $48），通过**极小仓位**控制了损失。

## 四、退出策略

### 4.1 100% 持有至结算

在全部 1,657 个仓位中：

- **卖出笔数：0**
- **所有仓位都持有到 5 分钟窗口结算**

这是最简单也最极端的退出策略——**完全不做止损、止盈或中途调整**。

### 4.2 为什么不卖？

1. **入场时已通过 BTC 偏离确认方向**——84%+ 胜率不需要中途调整
2. **仓位管理已内置风控**——错误仓位本身是小仓，全亏损失可控
3. **5 分钟窗口极短**——中途卖出的 spread 成本可能吞噬利润
4. **避免情绪化操作**——100% 持有到结算减少了决策点

### 4.3 退出策略的风险

- **无止损**意味着每个错误仓位都是**全额亏损**（Token 结算为 0，损失整个仓位成本）
- 这要求入场胜率必须足够高且仓位管理足够严格来覆盖亏损
- 计算：在入场价 0.65 时，胜率需 ≥ 65% 才能盈亏平衡（他在甜蜜区达到 82–86%）

## 5.5 Underdog（逆势彩票）策略

m101 在约 12%（106/845）的窗口中执行一种**独立的 underdog 策略**：以极低价格买入 BTC 反方向的 token，押注反转。**此策略严重依赖 maker 执行，taker 不可行。**

### 5.5.1 策略定义

| 特征 | 数值 |
|------|------|
| 窗口数（纯 underdog） | 106（12.5%） |
| 双向窗口中的彩票侧 | 另有 66 个窗口的次要方向 avg < 0.30 |
| 总计 | 172 个窗口涉及 underdog 操作 |
| 方向选择 | **83% 逆 BTC 当前趋势**（BTC 涨 → 买 Down，BTC 跌 → 买 Up）|
| 平均买入价 | 0.14（纯 underdog） |
| 胜率 | 21.2%（纯），19.4%（双向彩票侧） |
| ROI | **+37.5%**（纯），+47.1%（双向彩票侧） |
| 总 PnL | +$6,318（纯 underdog） |

### 5.5.2 触发条件分析

**高波动是必要条件，但不充分**：

| BTC 最大偏离 | 所有窗口 | 其中 underdog | 占比 |
|-------------|---------|-------------|------|
| ≥ $100 | 372 | 74 | 20% |
| ≥ $150 | 216 | 47 | 22% |
| ≥ $200 | 120 | 29 | 24% |
| ≥ $250 | 88 | 21 | 24% |

Underdog 窗口 BTC 最大偏离 avg=$199（vs 非 underdog avg=$114），但 80% 的高波动窗口 m101 仍选择跟随方向而非逆势。

**入场时 BTC 已偏离的幅度**：avg=$115，median=$94。m101 在 BTC 已大幅偏离后、判断可能过度延伸时入场。

### 5.5.3 投入与风控

| 指标 | Underdog | 非 Underdog |
|------|---------|-------------|
| 平均投入/窗口 | **$163** | $1,234 |
| 中位投入/窗口 | **$60** | $621 |
| 投入 < $100 | 63% | — |
| 投入 > $500 | 10% | — |

典型的**彩票注**模式：小额投入、高赔率。投入金额仅为正常窗口的 **1/8**。

### 5.5.4 入场时间 vs 价格

| 入场时间段 | 窗口数 | 平均买入价 | 平均投入 |
|-----------|--------|----------|---------|
| 0-60s | 24 | 0.215 | $395 |
| 60-120s | 14 | 0.167 | $177 |
| 120-180s | 16 | 0.109 | $129 |
| 180-240s | 12 | 0.107 | $105 |
| 240-300s | 12 | 0.075 | $56 |

越晚入场 → token 越便宜（因为 BTC 趋势在加强）→ 投入越小。这符合"越晚越确定方向，反转越不可能，所以投入越少"的逻辑。

### 5.5.5 Maker 优势在 Underdog 中尤为关键

m101 实际支付的价格比市场 ask 平均**便宜 $0.167**。在 avg ask 仅 0.31 的 underdog token 上，这意味着 maker 挂单拿到了**近乎半价**的成交。对于 taker 执行，underdog 策略的 EV 会大幅下降。

### 5.5.6 BTC 实际反转率

仅 **16%** 的 underdog 窗口中 BTC 最终反转（结算方向与入场时 BTC 趋势相反）。这与 21% 的胜率一致——underdog 策略本质是**低概率高赔率**：

- 价格 < 0.10：214% ROI（赔率 10:1，只需 >10% 胜率即正 EV）
- 价格 0.10-0.15：44% ROI
- 价格 0.20-0.30：**-22% ROI**（赔率不够高，负 EV）

### 5.5.7 对我们策略的启示

| 结论 | 说明 |
|------|------|
| ⚠️ **需要 Maker 才有 EV** | Taker 买入 underdog token 的成本比 m101 高约 50%，可能使正 EV 变为负 |
| ✅ **甜蜜区：price < 0.15** | 只有极低价格的 token 才有足够高的赔率覆盖低胜率 |
| ✅ **小额投入** | 每次 $50-100 的彩票注，不影响主策略资金 |
| ⛔ **暂不实现** | 依赖 maker 执行 + 未知的窗口选择信号，复制性低（⭐⭐）|

### 一句话总结

**m101 的策略极其简单：跟随 BTC 方向 + maker 挂单降低成本 + 高频参与。没有隐藏信号。**

### 完整交易流程

```

  │
  ├─ 观察 BTC 实时价格 vs 窗口开盘价
  │
  ├─ 方向判断：BTC > 开盘 → 买 Up，BTC < 开盘 → 买 Down
  │     └─ 92% 跟随 BTC 方向，8% 逆向（基本为 underdog 彩票）

  │     └─ 在 BTC 偏离 $30-60 的控制实验中，正确/错误预测的动量、一致性几乎相同
  │
  ├─ 分层 Maker 挂单（核心执行优势）
  │     └─ 每窗口 avg 44 笔限价买单，覆盖 88-133 秒
  │     └─ 挂单价格比 market ask 便宜 0.05-0.10（在低价 token 上优势更大）
  │     └─ 占总投入的 68%（$631K）
  │     └─ 这是 m101 入场价 0.66 vs 我们 0.84 的主要原因
  │
  ├─ Taker 补仓（确认性加仓）
  │     └─ taker 单笔更大（avg $61 vs maker $19），占 32%
  │     └─ 11.4% 的窗口纯 taker（紧急/高确定性场景）
  │
  ├─ DCA 加仓：36% 窗口有多次入场（中位间隔 18s，最多 8 次）
  │
  ├─ 约 12% 的窗口做 Underdog 彩票（详见 §5.5）
  │     └─ 逆 BTC 方向买入极低价 token（avg 0.14）
  │     └─ 小额投入（avg $163，仅正常窗口的 1/8）
  │     └─ 21% 胜率但 +37.5% ROI（低概率高赔率）
  │
  ├─ 连败风控：连续 3 败后仓位缩减 ~50%
  │
  └─ 100% 持有至结算（300s），零卖出，零止损
```

### 入场价真相（v3 修正）

v2 报告中的"avg 入场价 0.48"被 underdog 彩票单（avg 0.14）严重拉低：

| 范围 | 窗口数 | 平均入场价 | 投入 |
|------|--------|----------|------|
| 全部 | 899 | 0.575 | $929K |
| 去掉 underdog(<0.30) | 748 | **0.662** | $891K |
| 去掉 <0.50 | 561 | **0.742** | $730K |

m101 正常策略的真实入场价约 **0.66-0.74**，与我们的 0.84 差距约 **0.10-0.18**。

差距的构成：
- **Maker 挂单**：在 token ask 0.75 时，maker 可能以 0.65-0.70 成交 → 贡献 ~0.08
- **更早入场**：m101 avg 48-83s 入场（maker 单在等待成交），我们 deviation 触发 ~100s+ → 贡献 ~0.05
- **更低的入场门槛**：m101 在 BTC 偏离仅 $29-34 时就开始挂单 → 贡献 ~0.05

### 信号源分析（v3 新增）

**结论：m101 没有使用 BTC 价格之外的隐藏信号。**

验证方法：在控制 BTC 偏离（$30-60）后，对比 m101 正确与错误预测的窗口特征：

| 特征 | 正确预测 | 错误预测 | 差异 |
|------|---------|---------|------|
| BTC 30s 动量 | $27 | $27 | 0 |
| BTC 方向一致性 | 0.82 | 0.80 | ≈0 |
| 入场时间 | 137s | 109s | +28s |
| 投入金额 | $1,437 | $1,164 | +$273 |

动量、一致性完全相同 → 没有信号能区分对错。唯一差异是正确预测时入场稍晚、投入稍多，这可能只是"趋势延续时间越长越可能正确"的自然结果。

BTC 偏离阈值 vs 方向准确率 vs token ask 价格的关系也证实了这一点：

| 阈值 | 准确率 | token ask | E[ROI] |
|------|--------|-----------|--------|
| $20 | 68% | 0.70 | **-1.6%** |
| $40 | 77% | 0.78 | **-1.1%** |
| $60 | 85% | 0.85 | **-0.0%** |
| $80 | 89% | 0.89 | **+0.7%** |

**Polymarket 定价非常高效**：token 价格精确反映了当前 BTC 偏离下的真实概率。无论什么时间点以 taker 入场，E[ROI] 都接近零。这意味着：
- 降低 deviation 阈值不会改善 ROI
- 增加早期信号（imbalance 等）也不会有实质帮助
- **m101 的正 EV 几乎全部来自 Maker 执行**（以低于市场定价的价格成交）

### 盈利来源（按重要性排序）

1. **Maker 挂单执行（核心 alpha）**：以低于 market ask 0.05-0.10 的价格成交。在 Polymarket 近乎零和的定价中，这是唯一可持续的正 EV 来源。按 avg 0.08 maker 优势 × 845 窗口 × avg $1,100 投入估算，贡献约 $74K（占总 PnL $138K 的 54%）。
2. **Underdog 彩票策略（高赔率 alpha）**：106 个 underdog 窗口贡献 +$6,318，ROI +37.5%。严重依赖 maker 在极低价 token 上拿到近乎半价成交。
3. **高频参与 + 仓位管理**：845/约 6,000 个窗口参与（~14%），正确预测时投入略多（$1,433 vs $1,126）。但仓位差异不大（仅 1.3x），不是核心 alpha。
4. **BTC 动量持续性（市场基础）**：这不是 m101 特有的 alpha，而是整个 5 分钟市场的统计特性。所有跟随 BTC 方向的参与者都享有这个基础概率。

### 策略风险

1. **无止损**：错误仓位全额亏损，需持续保持 >60% 胜率
2. **利润集中度极高**：04-13 单日贡献 44% 总 PnL（$60,908），少数高波动日决定盈亏
3. **Maker alpha 可能被竞争侵蚀**：如果更多 maker 参与，成交价优势会收窄
4. **Polymarket 定价可能继续趋向高效**：如果 market maker 改进定价模型，token 价格与真实概率的偏差会更小
5. **样本有限**：仅 21 天数据

## 独立复核补充（基于 PG 交易数据 + btc_poly_1s_ticks）

> 以下内容为独立复核结论，与上文并列保留，重点回答：**从交易数据和市场数据看，m101 实际上在做什么。**

### 样本口径说明

- `m101_trades` 当前包含 **5m + 15m** 两类 BTC Up/Down 市场，不是纯 5m 样本
- 为了和 `btc_poly_1s_ticks` 对齐，以下判断主要基于 **5m 窗口**，且仅使用能成功对齐到 1s BTC 数据的样本
- 对齐后可直接复核的样本为：
  - 5m 交易：29,791 笔
  - 5m 窗口：647 个
  - 其中可与 `btc_poly_1s_ticks` 对齐的窗口：574 个

### 策略画像

**我的判断：m101 不是复杂预测模型，更像是一个“BTC 短线动量跟随 + maker 分层挂单降成本 + 少量反向彩票单”的高频执行策略。**

1. **主策略是顺着 BTC 当下方向做 5m Up/Down**

   在可对齐的 5m 样本中，排除低价彩票单后，m101 约 **88%** 的窗口是在跟随 BTC 当时相对开盘价的方向买入。  
   这更像是基于 **BTC 当下方向** 的顺势策略，而不是依赖复杂隐藏信号来预测结算方向。

2. **真正的优势更像执行，不像预测**

   m101 的成交明显以 **maker** 为主，再辅以少量 **taker** 补仓。  
   这说明它的核心不是等待一个特别神秘的入场点，而是先用分层挂单争取更低成本，再在需要时用 taker 快速补量。

3. **建仓方式是渐进式的，不是一次性重仓拍入**

   在可对齐的 5m 样本中，maker 成交笔数平均约 **43.2 笔/窗口**，中位数 **23 笔/窗口**。  
   这更像同方向的渐进式建仓：边观察、边挂单、边吸收流动性，而不是单次重仓入场。

4. **退出逻辑非常简单：基本持有到结算**

   数据中没有看到卖出行为，说明其风控更可能前置在 **入场方向、成交成本、仓位大小** 上，  
   而不是靠中途止损/止盈频繁调整。

5. **有一小部分是反向低价彩票单**

   在可对齐样本中，首单价格低于 `0.30` 的窗口有 **61 个**。这类窗口大多是**逆 BTC 当前方向**买入，  
   更像小仓位、高赔率的反转彩票，而不是主策略本身。

6. **仓位管理存在，但更像朴素经验而不是复杂模型**

   从整体行为看，m101 更像在做：  
   **强趋势/好价格就多做，弱信号或反向彩票就小做。**  
   这是一种很实战的资金分配方式，但不太像高度参数化的精密风险模型。

### 一句话总结

**m101 更像一个执行很强的顺势交易者，而不是一个靠复杂预测信号取胜的策略。**

其典型流程更接近：

`观察 BTC 当下方向 → 先挂便宜 maker 单拿成本 → 必要时用 taker 补仓 → 持有到结算 → 少量低价反转彩票单作为 side bet`

### 与上文结论的关系

- **可以保留的主线**：maker 为主、taker 为辅；BTC 方向跟随为主；少量 underdog；基本持有到结算
- **应更保守表述的地方**：
  - “92% 跟随 BTC”更适合保守写成 **约 88%**
  - “没有隐藏信号”属于**较强因果判断**，现有数据更稳妥的说法应是：**从已观察到的交易/市场数据看，没有看到必须引入隐藏信号才能解释其交易行为的证据**
